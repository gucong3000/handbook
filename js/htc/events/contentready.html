oncontentready事件

当附加有行为的元素其内容解析完成时，该事件发生。

格式

<PUBLIC:ATTACH
  event = "oncontentready"
  for = "sObject"
  id = "sID"
  onevent = "sEventHandler"
/>

属性

EVENT

必需。字符串值；指定动态HTML(DHTML)事件名称，或任何在HTC参考中列举出来的适用于HTML组件(HTC)的事件的名称。

FOR

可选。字符串值；用于指定事件源的下列值中的一个。

document document对象的引用。
element 默认值。附加有该行为的元素的引用。
window window对象的引用。
ID

可选。字符串值；在组件内用于识别该PUBLIC:ATTACH元素的唯一值。这个属性类似于DHTML中的ID属性。

ONEVENT

必需。字符串值；指定一段行内脚本或事件直接调用的处理函数。

注意

该元素的innerHTML属性在此事件发生前可能无法获得正确的值。

当PUBLIC:COMPONENT元素的literalContent设置为真且此方法发生时，该元素行为的固化内容保存在innerHTML属性中。此方法指明该元素行为可以访问innerHTML属性。

因为此方法为HTC组件独有，因此绑定该oncontentready事件的唯一途径是使用PUBLIC:ATTACH元素。attachEvent方法只作用于标准的DHTML事件，所以它并不能对此事件进行绑定。当一份HTC文件作为一个Viewlink来实现时，oncontentready事件或其他HTC独有的事件均无法在文档片段中附加到元素上。

要对一份HTC文件和它的主文档在初始化过程中事件的发生有一个大致了解，请阅读《组件初始化》。

范例

此例在一个页面上使用oncontentready方法来显示该元素的innerHTML属性的值。

<PUBLIC:ATTACH event="oncontentready" onevent="show_innerHTML()" />

<SCRIPT language="JScript">
function show_innerHTML()
{
  window.alert ('innerHTML = ' + element.innerHTML);
}
</SCRIPT>
