PUBLIC:COMPONENT元素

将文件内容识别为HTC。

格式

<PUBLIC:COMPONENT
  id = "sID"
  lightweight = "bLight"
  literalcontent = "sLiteral"
  name = "sName"
  supportseditmode = "bEditable"
  tagname = "sTagName"
  urn = "sURN"
>
<!-- child elements -->
</PUBLIC:COMPONENT>

属性

ID

可选。字符串值；在组件内用于识别该PUBLIC:COMPONENT元素的唯一值。这个属性类似于DHTML中的ID属性。

lightWeight

可选。布尔值；指定HTC文件中是否包含标记。如果HTC文件不包含标记，这个属性应该设置为true以改进渲染性能。这个属性对任何HTC文件均有效。

true 该HTC文件不包含标记，因此在主文档中使用该HTC文件定义的自定义标签时，无需每次都进行解析和渲染。
false 默认值。该HTC文件包含有标记，所以每次在主文档中使用该HTC文件定义的自定义标签时，都需要进行解析和渲染。
literalContent

可选。字符串值；指定包含在该自定义标签中的内容是作为数据岛处理，还是需要被解析并渲染。这个属性仅在该HTC文件定义了元素行为时才有效。此时同时还需要使用到tagName属性。阅读注意部分以获得更多的信息。

false 默认值。通过tagName属性定义的该标签中，文本和标记并不作为数据岛处理，必须被解析并渲染。
nested Internet Explorer 6以后继版本。通过tagName属性定义的该标签的第一个开启标记和最后一个关闭标记内的内容作为数据岛处理，并不被解析渲染。阅读使用literalContent属性以获得更多信息。
true 通过tagName属性定义的该标签的第一个开启标记和最后一个关闭标记内的内容作为数据岛处理，并不被解析渲染。
NAME

可选。布尔值；指定在包含有行为的文档中该行为的访问名称。

supportEditMode

可选。指定该HTC文件的内容是否可编辑的字符串。这个属性仅在该HTC文件定义了一个元素行为时才有效。

false 该HTC文件中的标记是可编辑的。
true 默认值。该HTC文件中的标记是不可编辑的。你可以在该HTC文件中的每个元素上通过设置isContentEditable属性为VARIANT_TRUE来重载此值。
tagName

可选。字符串值；指定该HTC文件定义并导入到主文档中的自定义标签的名称。此属性仅当该HTC文件定义了一个元素行为时才有效。阅读注意部分以获得更多信息。

URN

可选。字符串值；符合统一资源名称(URN)格式的唯一值，用于识别该组件。此属性让多个行为发生同名事件时能够识别出各个组件。当事件发生时，event对象的srcUrn属性被设置为发生事件的行为的URN。

元素信息

子元素 最多一次。PUBLIC:DEFAULTS
一次或多次。PUBLIC:ATTACH，PUBLIC:EVENT，PUBLIC:METHOD，PUBLIC:PROPERTY
最低可用版本 Internet Explorer 5
最低操作系统版本 Windows 95，Windows NT 4.0

注意

该PUBLIC:COMPONENT元素用于定义良种截然不同类型的行为。第一种是附加性行为，它使用behavior层叠样式表(CSS)属性来修改一个已存在元素的行为。第二种类型从Internet Explorer 5.5开始引入，被称为元素行为。元素行为用于定义一个自定义标签，然后如一个标准的HTML标签一样在网页中使用。

元素行为暗示着在网页中一个自定义标签被创建然后适用。因此，有必要在该HTC文件定义一个元素行为时定义PUBLIC:COMPONENT元素的tagName属性。正如前面所说的，此元素的一些属性明确为元素行为的使用而设计。要获得更多的信息，请阅读《元素行为》。

NAME属性在多个行为附加到一个元素上时特别有用，因为它让你可以在包含文档中调用所需行为的属性或者方法。如果在一个文档中多个行为被附加到<SPAN id="mySpan" />元素上，你可以使用下面的格式来设置命名为behaviorABC的行为的delay属性。

mySpan.behaviorABC.delay = 1000;
