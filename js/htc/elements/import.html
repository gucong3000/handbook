IMPORT处理指令

从元素行为中导入标签的定义。

格式

<?IMPORT
  namespace = "sNameSpace"
  implementation = "sImplementation"
/>

属性

namesapce

必需。字符串值；指定标签定义要导入的一个已定义的命名空间。

implementation

必需。下列值之一：

#default Internet Explorer 5.5及后继版本中浏览器内嵌的元素行为。
#objectID 字符串值；指定object标签的id属性。
SImplementation 字符串值；指定包含标签定义的HTML组件(HTC)文件。

处理指令信息

出现次数 无限制
父元素 HEAD，BODY
最低可用版本 Internet Explorer 5.5
最低操作系统版本 Windows 95，Windows NT 4.0，Windows CE 4.0

注意

一个页面中可以使用多次IMPORT处理指令。

HTML标签的XMLNS属性必须符合IMPORT处理指令定义的namespace属性值。否则导入的标签定义无法正常使用。

导入的标签如果implementation属性值无效将无法被渲染。如果你使用一份HTC文件，则它必须遵循所有行为都相同的安全准则。欲了解安全方面的更多信息，请阅读《DHTML行为简介》。

IMPORT处理指令可以放置在文档正文结束前的任何地方。但是，导入某个自定义标签的IMPORT处理指令必须放置在该标签第一次被使用之前。如果IMPORT处理指令放置在自定义元素之后，行为会无法附加到自定义元素上。这也意味着document.write方法无法用于向文档中添加IMPORT处理指令。

作为IMPORT处理指令的替代方法，使用doImport方法来动态导入元素行为。doImport方法用于与createElement方法联合添加附加了元素行为的自定义元素。

IMPORT处理指令只在文档的初始化解析过程中才会被处理。因此使用了元素行为的文档必须在HTML文件中包括相应的IMPORT处理指令。当主文档中已经指定了IMPORT处理指令之后，document.write方法可以用于添加自定义元素及附加行为。

IMPORT处理指令不是all集合的成员之一。但是其内容可以被document.body.innerHTML属性访问到。

范例

下面的样例演示了IMPORT处理指令的用法。HTML标签中定义的XMLNS属性用于匹配IMPORT处理指令namespace属性。

<HTML xmlns:games>
<?IMPORT namespace="games" implementation="checkers.htc" />
<BODY>
<games:checkers />
</BODY>
</HTML>

没有应用于此处理指令的公共标准。
